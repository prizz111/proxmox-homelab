# 03.1 — Ubiquiti UniFi Flex Mini Switch: Initial Setup

Set up your new UniFi Flex Mini switch from scratch and configure it to work with your AT&T home network and Proxmox cluster.

> **Read this before [03 - Network Configuration](03-network-config.md).** This guide gets the switch online and ready; guide 03 uses it to configure Proxmox VLANs.

---

## Why This Matters

The UniFi Flex Mini is a **managed switch** — unlike a basic "dumb" switch that just passes traffic, a managed switch lets you create VLANs, monitor traffic per-port, and control how devices communicate. That capability is what allows you to separate Proxmox cluster traffic onto its own VLAN.

The trade-off is that UniFi switches require a **UniFi Network controller** — a piece of software — to configure them. You can't just plug them in and use a browser to hit the switch's IP like some other brands. The controller can run on your PC, in the cloud, or (later) on your Proxmox cluster itself.

---

## What You Need

- UniFi Flex Mini switch (out of the box)
- AT&T gateway (BGW320, BGW210, or similar — your AT&T-provided router/modem)
- 2–3 Ethernet cables
- A Windows, Mac, or Linux PC **temporarily** (for setup)
- A free Ubiquiti account at [account.ui.com](https://account.ui.com)

---

## Step 1: Physical Connection

**Before powering the switch on**, understand the port layout of the UniFi Flex Mini:

```
[Port 1]  [Port 2]  [Port 3]  [Port 4]  [Port 5]
                                          ↑
                                    Uplink / PoE in
                                 (connect to AT&T router)
```

- **Port 5** is the designated uplink port — connect this to one of the LAN ports on your AT&T gateway
- **Ports 1–4** are for your devices (the 3 Proxmox nodes will go here)

> **Power**: The Flex Mini is powered via **PoE on Port 5** (requires a PoE injector or PoE-capable upstream device), **or** via a **USB-C power adapter** on the side. Your AT&T gateway likely doesn't supply PoE, so use the included USB-C adapter to power the switch.

**To connect:**

1. Plug the USB-C power adapter into the switch and into a wall outlet
2. Connect an Ethernet cable from your **AT&T gateway's LAN port** → **Flex Mini Port 5**
3. The switch LEDs will blink as it boots (takes ~30 seconds)

At this point the switch is online but unconfigured. It's waiting to be "adopted" by a UniFi Network controller.

---

## Step 2: Set Up UniFi Network Controller

UniFi switches must be managed through the **UniFi Network application**. You have two options:

### Option A: UniFi Network on Your PC (Easiest to Start)

Install the UniFi Network application on your PC temporarily. Once the switch is configured, the controller just needs to be running when you want to make changes — it doesn't need to run 24/7 for the switch to function.

1. Go to [ui.com/download/network-server](https://ui.com/download/network-server)
2. Download **UniFi Network Server** for Windows
3. Install and open it — it will launch in your browser at `https://127.0.0.1:8443`

> The installer also installs a Java dependency (MongoDB) — this is normal.

### Option B: UniFi Cloud (No Local Install)

Ubiquiti offers cloud-hosted controller management. The switch will "phone home" to Ubiquiti's servers and you manage it via browser at [unifi.ui.com](https://unifi.ui.com).

- **Pros**: No software to install, accessible from anywhere
- **Cons**: Requires active internet; some features lag behind local controller

For initial setup, either option works. This guide uses **Option A (local)** as it's more reliable for first-time setup.

---

## Step 3: Create a UniFi Account and First-Run Setup

When you open UniFi Network in your browser:

1. Sign in with your Ubiquiti account (create one at [account.ui.com](https://account.ui.com) if needed)
2. The setup wizard will ask:
   - **Setup type**: Select **Advanced Setup** (gives you more control)
   - **Country**: United States
   - **Timezone**: Your timezone

You'll land on the main UniFi Network dashboard. It looks empty — that's expected.

---

## Step 4: Adopt the Switch

The Flex Mini on your network will be discovered automatically.

1. In the UniFi Network dashboard, look for a **notification** or check **Devices** in the left sidebar
2. You should see the **UniFi Flex Mini** listed with status **"Pending"**
3. Click on it → click **Adopt**

The switch will go through these states:
```
Pending → Adopting → Provisioning → Connected
```

This takes about 1–3 minutes. Once it shows **Connected** with a green dot, the switch is under your control.

> **If the switch doesn't appear**: Make sure your PC is on the same network (connected to the AT&T gateway via WiFi or Ethernet). The controller discovers devices using layer-2 broadcast — it only works on the same local network.

---

## Step 5: Rename the Switch and Ports

Before configuring VLANs, label things clearly.

1. Click on the switch in the Devices list → click the **Settings** (gear) icon
2. Set the **Alias** (name) to: `homelab-switch`
3. Under **Port Management**, rename each port:
   - Port 1 → `pve1`
   - Port 2 → `pve2`
   - Port 3 → `pve3`
   - Port 4 → `spare`
   - Port 5 → `uplink-att`
4. Click **Apply Changes**

---

## Step 6: Create VLAN 10 (Proxmox Cluster Network)

Now create the VLAN that will carry Proxmox cluster traffic. In UniFi, VLANs are managed through **Networks**.

1. Go to **Settings** (gear icon, left sidebar) → **Networks**
2. Click **Create New Network**
3. Fill in the form:

| Field | Value |
|---|---|
| Name | `proxmox-cluster` |
| Purpose | `VLAN Only` |
| VLAN ID | `10` |
| Network Type | `VLAN Only` (no DHCP — cluster IPs are static) |

4. Click **Add Network**

> **"VLAN Only"** means UniFi won't try to assign IPs or run DHCP on this VLAN — that's correct, because the Proxmox nodes use manually configured static IPs on this VLAN (`10.10.10.x`).

---

## Step 7: Create a Port Profile for Trunk Ports

The 3 Proxmox nodes need their switch ports to carry **both** VLAN 1 (untagged, management traffic) **and** VLAN 10 (tagged, cluster traffic) on a single cable. This is called a **trunk port**.

In UniFi, you configure this through a **Port Profile**.

1. Go to **Settings** → **Profiles** → **Switch Ports**
2. Click **Create New Port Profile**
3. Configure it:

| Field | Value |
|---|---|
| Name | `proxmox-trunk` |
| Native Network | `Default` (VLAN 1 — your home network) |
| Tagged Networks | `proxmox-cluster` (VLAN 10) |

4. Click **Save**

This profile says: "send untagged VLAN 1 traffic and tagged VLAN 10 traffic over this port."

---

## Step 8: Apply the Port Profile to Proxmox Ports

Now apply the `proxmox-trunk` profile to ports 1, 2, and 3 (the ports your Proxmox nodes will plug into).

1. In **Devices**, click on `homelab-switch`
2. Go to the **Ports** tab
3. Click on **Port 1** (pve1):
   - Change **Profile** from `All` to `proxmox-trunk`
   - Click **Apply**
4. Repeat for **Port 2** (pve2) and **Port 3** (pve3)
5. Leave Port 5 (uplink-att) on its default profile — it stays as a regular untagged access port

The switch will briefly reprovision each port as you apply the profile.

---

## Step 9: Verify the Physical Setup

Connect your 3 Proxmox nodes to the switch:

| Switch Port | Connect To |
|---|---|
| Port 1 | pve1 (Ethernet on HP EliteDesk) |
| Port 2 | pve2 |
| Port 3 | pve3 |
| Port 5 | AT&T Gateway LAN port |

Check that the LEDs for ports 1–3 light up (solid or blinking) when the Proxmox nodes are powered on. In the UniFi dashboard, the Devices page will show the switch and its connected clients.

---

## Understanding Your AT&T Setup

Your AT&T gateway (BGW320 or similar) is both a modem and router. This means:

```
[AT&T Fiber] → [AT&T BGW320 Gateway] → [UniFi Flex Mini] → [Proxmox Nodes]
                  192.168.1.254                                192.168.1.11–13
```

- The AT&T gateway manages your internet connection and runs DHCP for your home network (`192.168.1.x`)
- The UniFi switch sits between the gateway and your Proxmox nodes — it passes VLAN 1 traffic through normally (so nodes get internet access) and also carries VLAN 10 cluster traffic between nodes
- VLAN 10 traffic (`10.10.10.x`) **stays on the switch** — it never reaches the AT&T gateway

> **Note on AT&T IP Passthrough**: AT&T gateways have strict NAT and firewall settings. You don't need to change anything on the AT&T gateway for the Proxmox cluster to work — all cluster and management traffic stays within your local LAN.

---

## Optional: Keeping the Controller Running

If you used Option A (local PC controller), the switch works fine even when the controller is off — all your VLAN and port configurations are **stored on the switch itself**. You only need the controller running when you want to make changes.

When you eventually move the controller to your Proxmox cluster (a common next step), you can run it as a Docker container or LXC and have it available 24/7.

- [ ] UniFi Network controller installed and running on PC

- [ ] Switch adopted and showing **Connected** in UniFi
- [ ] Switch renamed to `homelab-switch`, ports labeled
- [ ] Network `proxmox-cluster` created (VLAN ID: 10, VLAN Only)
- [ ] Port profile `proxmox-trunk` created (Native: Default, Tagged: proxmox-cluster)
- [ ] Ports 1, 2, 3 have `proxmox-trunk` profile applied
- [ ] Proxmox nodes physically connected to ports 1, 2, 3
- [ ] All 3 node ports show as connected/active in UniFi dashboard
